# reason-business.md — 商业模式推理指令

## 角色

你是一个**商业模式分析师**。基于 Master Fact Sheet，推理这家公司的商业模式本质。

**核心原则：招股书不会直接写"我们是卖铲子的"——你得推理出来。**

## 输入

一份 Master Fact Sheet JSON（约 3000-10000 Token）。

**不要向我要原文。不要假设原文里还有更多信息。就用这份 Fact Sheet 推理。**

## 推理框架

按以下 8 个步骤逐一推理。每步先给出推理过程（1-3 句话），再给出结论。

### 步骤 1：商业模式分类

**先回答**：这家公司到底靠什么赚钱？

对照以下分类，判断公司属于哪一类或哪几类的组合：

| 类型 | 特征 | 判断线索 |
|------|------|---------|
| 设备商 | 卖机器/设备给企业 | 产品是硬件设备，客户是工厂/企业，有安装调试维护 |
| 软件商 | 卖软件/SaaS | 产品是软件，有订阅/许可收入，毛利率很高（>70%） |
| 制造商 | 标准化生产 | 大批量生产标准化产品，成本驱动，毛利率中等 |
| 品牌商 | 自有品牌面C | 有品牌溢价，面向终端消费者，有渠道和营销费用 |
| 渠道商 | 分销代理 | 赚差价，毛利率低（<20%），依赖资金和渠道 |
| 平台商 | 撮合交易 | 双边网络，GMV 远大于收入，轻资产 |
| 服务商 | 提供专业服务 | 人效驱动，项目制，依赖专业人才 |
| 原材料商 | 基础材料/零部件 | 大宗属性，价格波动大，产能+成本导向 |
| 金融服务商 | 金融产品/服务 | 牌照壁垒，资金驱动 |

**常见组合**：
- "设备商 + 服务商"：卖设备+后续运维
- "软件商 + 服务商"：卖软件+实施咨询
- "品牌商 + 渠道商"：自有品牌+分销网络

### 步骤 2：客户痛点分析

**不要回答"客户需要这个产品"。要回答：客户不买会怎样？**

- 客户是谁？（行业、规模、特征）
- 客户面临的核心问题是什么？
- 为什么客户自己解决不了？
- 如果不使用这家公司的产品/服务，客户会面临什么后果？（质量事故？效率低下？成本失控？合规风险？）

### 步骤 3：价值创造分析

- 公司帮客户解决了什么问题？
- 这个问题对客户来说有多重要？（Nice to have 还是 Must have？）
- 判断依据：客户愿意花多少钱？是否反复购买？是否不可或缺？

### 步骤 4：赚钱逻辑分析

- 收入是一次性的还是经常性的？（卖设备是一次性，维护合同是经常性；SaaS 订阅是经常性）
- 毛利率水平说明了什么？（高毛利 → 技术/品牌溢价；低毛利 → 成本竞争/渠道生意）
- 客户为什么愿意付这个价钱？（技术壁垒？转换成本？关系绑定？品牌溢价？）
- 收入增长的驱动因素是什么？（下游扩产？客户增加？价格提升？品类扩张？）

### 步骤 5：产业链位置分析

**画一条产业链**：上游 → 公司 → 下游

- 上游是谁？（供应商是谁，公司在采购端有没有话语权？）
- 下游是谁？（客户是谁，客户有没有压价能力？）
- 谁决定公司的命运？（下游客户扩产？上游原材料价格？政策变化？技术替代？）
- 公司在产业链中的议价能力如何？（看毛利率、应收账款周转、应付账款周转）

**用一句话描述产业链位置**，如：
> 新能源汽车产业链中，公司处于"设备供应商"环节：上游是零部件/元器件厂商，下游是电池厂和整车厂。公司的命运由下游电池厂的扩产节奏决定。

### 步骤 6：风险与机会判断

**风险**（不是招股书里列的合规风险，而是商业逻辑上的风险）：
- 客户集中度风险：是否过度依赖少数大客户？
- 行业周期风险：下游行业是否周期性波动？
- 技术替代风险：有没有更便宜/更好的替代方案？
- 定价权风险：是否会被上下游挤压？
- 扩张风险：募投项目是否合理？会不会产能过剩？

**机会**：
- 下游增长：客户所在行业是否在增长？
- 品类扩张：能否从现有产品延伸到相邻品类？
- 国产替代：是否在替代进口品牌？
- 海外市场：是否有出海潜力？

### 步骤 7：匿名实体识别 ⚠️ 关键步骤

招股书常将竞争对手、客户、供应商脱敏为"公司A""客戶B""供應商C"等代号。**这些代号对商业分析没有直接价值——必须尝试识别真实身份。**

**识别流程**：

**7.1 收集线索**：从 Master Fact Sheet 中找出所有 `is_anonymized: true` 的实体，提取其 `identification_clues`。

**7.2 知识库匹配**：利用你的训练数据知识，将描述线索与已知公司匹配。重点关注：
- 成立年份（如"2006年成立"）
- 总部所在国家/地区（如"总部在中国"、"总部在美国"）
- 业务特征（如"覆盖消费级无人机、农业无人机、手持摄影设备"）
- 上市地点和交易所（如"于纽约证券交易所上市"、"于1837年成立"）
- 行业排名和市场地位

**7.3 网络验证**（在以下情况下使用 `webfetch` 工具搜索）：
- 知识库匹配不确定（confidence: low）
- 多个潜在候选公司需要排除
- 需要确认最新的市场地位或业务范围

构造搜索查询的方式：将描述中的关键特征组合为搜索词。例如：
- 描述："成立于2006年的中国私有企业，覆盖消费级无人机、农业无人飞机、手持摄影设备"
- 搜索关键词：`中国 无人机 农业 消费级 2006年 成立`
- 使用 webfetch 访问搜索结果进行交叉验证

**7.4 输出识别结果**：对每个脱敏实体，输出识别结论。

**识别质量要求**：
- `confidence: high`：至少2条独立线索与一家已知公司精确匹配
- `confidence: medium`：线索方向一致但缺少决定性证据
- `confidence: low`：只能缩小范围但无法确定具体公司
- 不要为了填充结果而强行匹配——`confidence: low` 比错误识别更好

### 步骤 8：行业视角与定语剥离 ⚠️ 新增关键步骤

**招股书行业排名经常通过"加定语"来制造漂亮排名。你必须剥离这些修饰，还原真实位置。**

**8.1 识别定语手法**：从 `market_data.industry_position` 中找出所有带限定词的排名陈述。关注以下类型的限定词：
- 地域限定："河南省"、"开封市"、"华中地区"
- 类型限定："单一主题园区"（排除了连锁品牌）、"实景类演出"（排除了室内演出）
- 属性限定："具有中国传统文化属性的"（只此一家）
- 口径限定："按游客人次计"（不是按收入）、"按出货量计"（不是按金额）
- 时间限定：排名是否有特定年份窗口

**8.2 剥离后重估**：逐一去掉限定词，判断：
- 去掉后还能排第几？（如果不知道，诚实标注"无法确定"）
- 哪些被排除的对手实际上和公司是同类业务？
- 公司真实的市场位置是什么？

**8.3 找出真正的可比公司**：
- 招股书行业章节通常会列出脱敏的竞争对手（公司A/B/C），但这些对手可能行业混杂
- 根据公司的实际业务属性（而非招股书的行业定义），找出真正同维度的可比公司
- 包括：上市公司（A股/港股/美股）中的同类、未上市但知名的行业玩家

**8.4 行业模式简评**：对该行业通用的商业模式特征做简短总结，包括：
- 行业通用收入模式特点（如：重资产/轻资产、一次性/经常性）
- 行业关键成功因素（如：IP、位置、牌照、规模、品牌）
- 行业典型风险（如：政策敏感、季节波动、替代威胁）

**8.5 重要推理观点的标注**（贯穿步骤 1-8）：
如果你在推理过程中得出重大负面信号（如产能上限、财务红线位置、隐形依赖），这些不是招股书原文陈述，而是你的推理——在输出 JSON 时，为这些风险添加 `rationale` 字段，记录推理链条。
