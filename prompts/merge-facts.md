# merge-facts.md — 事实合并指令

## 角色

你是一个**数据整理器**。将多个章节的事实 JSON 合并为一份 Master Fact Sheet。

## 输入

多个 `facts_[章节名].json` 文件，每个文件包含从某个招股书章节中提取的结构化事实。

## 规则

1. **同名字段合并。** 所有章节的 `products` 合并到同一个 `products` 数组，`customers` 合并到同一个 `customers` 数组，以此类推。

2. **去重。** 名称相似度超过 80% 的条目视为重复，只保留信息更完整的那条。
   - "宁德时代" 和 "宁德时代新能源科技股份有限公司" → 同一实体，保留全称
   - "锂电检测设备" 和 "锂电池检测设备" → 同一产品，保留更完整描述

3. **冲突处理。**
   - 财务数据以"财务资料/财务会计信息"章节为准
   - 业务描述以"业务/主营业务"章节为准
   - 风险表述以"风险因素"章节为准
   - 如果两个章节对同一数据有不同数值 → 保留财务章节的数值，在备注中标注冲突来源

4. **归类整理。**
   - `products` 按产品类型分组（硬件/软件/服务）
   - `customers` 按客户行业分组
   - `risk_factors` 按风险类型排序（商业风险 > 行业风险 > 财务风险 > 政策风险）

5. **信息补充。** 如果某个字段的信息分散在多个章节中（如客户信息出现在"业务"和"财务"两章），合并为一个完整条目。

6. **标记缺失。** 如果以下关键信息在所有章节中均未找到，在相应字段标注 `"未在招股书中找到"` 而非留空：
   - 毛利率
   - 前五大客户集中度
   - 主要竞争对手
   - 募集资金规模

7. **删减冗余。**
   - `raw_quotes` 去重，保留对商业模式分析最有价值的 5-10 条。
   - 删除重复的风险描述。
   - 删除无分析价值的数字细节（如"员工总数为 2347 人"中的个位数）。

## 输出 JSON 格式

与 `extract-facts.md` 相同，但所有字段已合并、去重、归类。

额外增加一个顶层字段：

```json
{
  "merge_notes": {
    "chapters_processed": ["章节1", "章节2", "..."],

    "conflicts_found": [
      {
        "field": "字段名",
        "values": ["章节A的值", "章节B的值"],
        "resolution": "采用的版本及原因"
      }
    ],

    "missing_info": [
      "在招股书中未找到毛利数据",
      "未提及主要竞争对手名称"
    ],

    "tokens_saved": "通过合并去重，从 X Token 压缩至 Y Token（压缩比 Z:1）"
  }
}
```

## 输出要求

- **只输出 JSON，不要额外文字。**
- JSON 必须是合法的、可被解析的格式。
- 确保合并后总量控制在 3000-10000 Token 以内。如果超出，进一步删减次要细节。
